## 这版解决的问题

1. 默认从“扫到即停并重开摄像头”调整为“跟手优先”。
2. 接收端默认相机常开，只做低频识别，减少反复打开摄像头导致的黑屏、重新对焦和不跟手。
3. 发送端默认播放间隔调整为 600ms。
4. 接收端默认接收冷却调整为 500ms，比发送端短约 100ms，便于提前进入下一片识别窗口。
5. 单片重启兜底模式保留，默认重启延迟调为 300ms。
6. 摄像头停止增强：捕获 getUserMedia 返回的 stream，停止时统一 stop 所有 tracks。
7. 接收端默认更轻：fps=1、resolution=360、zoom=1.0、日志最多 30 条。

## 推荐默认参数

```text

```

接收端：

```text
模式：跟手优先：相机常开低频识别
FPS：1
分辨率：360
缩放：1.0
接收冷却：500ms
```

## 调参建议

如果还是卡顿：

```text
保持跟手优先
FPS 保持 1
分辨率保持 360
发送端播放间隔调到 800ms
接收冷却调到 700ms
```

如果漏扫明显：

```text
发送端播放间隔 3000 → 3500ms
接收冷却 500 → 400ms
分片大小 1200 → 900
```

如果希望进一步省资源：

```text
切到“省电防卡：扫到即停并自动重开”
单片重启默认 300ms
```
